Xác minh Team và Dataset Setup

Trong bước này, bạn sẽ xác minh rằng tất cả các thành phần team và dataset đã được thiết lập đúng cách.

1. Kiểm tra Glue Databases

  1. Vào AWS Glue Console
  2. Click Data Catalog → Databases
  3. Xác nhận có các databases:
    • sdlf-workshop-team
    • sdlf-workshop-dataset

2. Kiểm tra Glue Tables

  1. Chọn database sdlf-workshop-team
  2. Click Tables
  3. Xác nhận có table team với data
  4. Lặp lại cho database sdlf-workshop-dataset

3. Kiểm tra S3 Buckets

  1. Vào S3 Console
  2. Xác nhận có các buckets:
    • sdlf-workshop-team-data
    • sdlf-workshop-raw-bucket
  3. Kiểm tra folder structure trong raw bucket

4. Kiểm tra IAM Roles

  1. Vào IAM Console
  2. Click Roles
  3. Tìm và xác nhận role sdlf-workshop-team-role
  4. Kiểm tra attached policies

5. Kiểm tra IAM Policies

  1. Click Policies
  2. Tìm và xác nhận policies:
    • sdlf-workshop-team-policy
    • sdlf-workshop-dataset-policy
  3. Kiểm tra policy content

6. Kiểm tra User Group

  1. Click Groups
  2. Tìm group sdlf-workshop-team-group
  3. Xác nhận attached policies
  4. Kiểm tra group members

7. Kiểm tra Lake Formation Permissions

  1. Vào AWS Lake Formation Console
  2. Click Data permissions
  3. Kiểm tra permissions cho:
    • Database sdlf-workshop-team
    • Role sdlf-workshop-team-role

8. Test Athena Query

  1. Vào Amazon Athena Console
  2. Chọn Workgroup: sdlf-workshop-workshop
  3. Chọn Database: sdlf-workshop-team
  4. Chạy test query: 
    SELECT * FROM team LIMIT 10;

9. Test S3 Access

  1. Vào S3 Console
  2. Chọn bucket sdlf-workshop-team-data
  3. Thử upload một file test
  4. Xác nhận có thể upload và download

10. Test Glue Access

  1. Về AWS Glue Console
  2. Chọn database sdlf-workshop-team
  3. Thử tạo một table test
  4. Xác nhận có thể tạo và xóa table

11. Checklist cuối cùng

  • ✅ Glue databases đã tạo
  • ✅ Glue tables có data
  • ✅ S3 buckets đã tạo
  • ✅ IAM roles có đủ permissions
  • ✅ IAM policies đã tạo
  • ✅ User group đã tạo
  • ✅ Lake Formation permissions đã cấu hình
  • ✅ Athena queries hoạt động
  • ✅ S3 access hoạt động
  • ✅ Glue access hoạt động

12. Troubleshooting

Nếu gặp vấn đề:

  1. Access Denied: Kiểm tra IAM permissions
  2. Table not found: Kiểm tra Glue crawler
  3. Bucket not accessible: Kiểm tra S3 bucket policy
  4. Query failed: Kiểm tra Athena workgroup

Lưu ý:

  • Ghi lại tất cả resource names để sử dụng cho các bước tiếp theo
  • Test tất cả permissions trước khi chuyển sang phần tiếp theo
  • Nếu có lỗi, kiểm tra CloudWatch logs

13. Chuẩn bị cho phần tiếp theo

Sau khi hoàn thành xác minh:

  1. Ghi lại team_id: workshop-team-001
  2. Ghi lại dataset_id: workshop-dataset-001
  3. Ghi lại bucket names và database names
  4. Chuẩn bị cho phần ETL Pipeline

Đến đây bạn đã hoàn thành việc thiết lập Team và Dataset. Tiếp theo chúng ta sẽ triển khai ETL Pipeline.
