Configure Permissions

Configure Permissions for Team and Dataset

In this step, we will configure access permissions for teams and datasets in SDLF.

1. Access IAM Console

  1. Login to AWS Console
    • Open AWS Console
    • Search for IAM in the service search bar
    • Click on IAM

2. Create IAM Role for Team

  1. Click Create role

    • Trusted entity: AWS service
    • Service: Lambda
    • Click Next

  2. Attach policies

    • Find and select necessary policies:
      • AmazonS3FullAccess
      • AWSGlueServiceRole
      • AmazonAthenaFullAccess
    • Click Next

  3. Configure role

    • Role name: sdlf-team-role
    • Description: Role for SDLF team access
    • Click Create role

3. Create IAM Policy for Dataset

  1. Click Create policy
    • Select JSON tab
    • Add policy document:
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "s3:DeleteObject"
      ],
      "Resource": "arn:aws:s3:::sdlf-workshop-bucket/*"
    },
    {
      "Effect": "Allow",
      "Action": [
        "glue:GetTable",
        "glue:GetTables",
        "glue:GetDatabase",
        "glue:GetDatabases"
      ],
      "Resource": "*"
    }
  ]
}
  1. Save policy
    • Policy name: sdlf-dataset-policy
    • Description: Policy for dataset access
    • Click Create policy

4. Attach Policy to Role

  1. Select the created role

    • Click on role sdlf-team-role

  2. Attach policy

    • Click Attach policies
    • Find and select sdlf-dataset-policy
    • Click Attach policy

5. Verify Configuration

  1. Check role

    • Go to Roles in IAM
    • Confirm role has sufficient policies

  2. Test permissions

    • Use AWS CLI or Console to test access permissions

Make sure to follow the principle of least privilege when configuring permissions.
