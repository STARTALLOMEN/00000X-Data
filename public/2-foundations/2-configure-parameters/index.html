Configure Parameters

Configure Parameters

Overview

In this step, we will configure the parameters required for SDLF foundations deployment. These parameters define how your data lake will be structured and secured.

SDLF Configuration Files

SDLF uses several configuration files to customize the deployment:

  • parameters.json - Main configuration parameters
  • config.json - Environment-specific settings
  • team-config.json - Team and dataset configurations

Step 1: Navigate to Foundations Directory

cd ~/sdlf-workshop/aws-serverless-data-lake-framework/sdlf-foundations
ls -la

You should see the CloudFormation templates and parameter files.

Step 2: Configure Main Parameters

Edit parameters.json

cp parameters-example.json parameters.json

Edit the parameters.json file:

{
    "pOrganizationName": "MyOrg",
    "pApplicationName": "sdlf",
    "pEnvironment": "dev",
    "pTeamName": "demo",
    "pDatasetName": "legislators",
    "pKMSInfraKeyId": "",
    "pKMSDataKeyId": "",
    "pEnableVpc": "false",
    "pVpcCidr": "10.0.0.0/16",
    "pEnableNat": "false",
    "pTagEnvironment": "dev"
}

Parameter Descriptions

Parameter Description Example Value
pOrganizationName Your organization name MyOrg
pApplicationName Application prefix sdlf
pEnvironment Environment (dev/test/prod) dev
pTeamName Team name for this deployment demo
pDatasetName Dataset name legislators
pKMSInfraKeyId KMS key for infrastructure (optional)
pKMSDataKeyId KMS key for data (optional)
pEnableVpc Enable VPC deployment false
pVpcCidr VPC CIDR block 10.0.0.0/16
pEnableNat Enable NAT Gateway false
pTagEnvironment Environment tag dev

Step 3: Configure Environment Settings

Create config.json

cat > config.json << EOF
{
    "dev": {
        "account": "$(aws sts get-caller-identity --query Account --output text)",
        "region": "$(aws configure get region)"
    }
}
EOF

Verify configuration

cat config.json

Expected output:

{
    "dev": {
        "account": "123456789012",
        "region": "us-east-1"
    }
}

Step 4: Set Environment Variables

# Export required variables
export ORGANIZATION_NAME="MyOrg"
export APPLICATION_NAME="sdlf"
export ENVIRONMENT="dev"
export TEAM_NAME="demo"
export DATASET_NAME="legislators"

# Verify variables
echo "Organization: $ORGANIZATION_NAME"
echo "Application: $APPLICATION_NAME"
echo "Environment: $ENVIRONMENT"
echo "Team: $TEAM_NAME"
echo "Dataset: $DATASET_NAME"

Step 5: Validate Configuration

Run validation script

# Check if validation script exists
if [ -f scripts/validate-config.sh ]; then
    ./scripts/validate-config.sh
else
    echo "Manual validation required"
fi

Manual validation

# Check AWS account and region
aws sts get-caller-identity
aws configure get region

# Validate S3 bucket names will be unique
echo "Raw bucket: $APPLICATION_NAME-$TEAM_NAME-$DATASET_NAME-raw-$ENVIRONMENT"
echo "Stage bucket: $APPLICATION_NAME-$TEAM_NAME-$DATASET_NAME-stage-$ENVIRONMENT"
echo "Analytics bucket: $APPLICATION_NAME-$TEAM_NAME-$DATASET_NAME-analytics-$ENVIRONMENT"

Step 6: Review Resource Names

Based on your configuration, SDLF will create resources with these naming patterns:

S3 Buckets

  • Raw Data: sdlf-demo-legislators-raw-dev-[account-id]
  • Stage Data: sdlf-demo-legislators-stage-dev-[account-id]
  • Analytics Data: sdlf-demo-legislators-analytics-dev-[account-id]
  • Artifacts: sdlf-artifacts-dev-[account-id]

IAM Roles

  • Team Role: sdlf-demo-team-role-dev
  • Pipeline Role: sdlf-demo-legislators-pipeline-role-dev
  • Glue Role: sdlf-demo-legislators-glue-role-dev

KMS Keys

  • Infrastructure Key: alias/sdlf-infra-key-dev
  • Data Key: alias/sdlf-data-key-dev

Step 7: Create Deployment Script

Create a deployment script for easy execution:

cat > deploy-foundations.sh << 'EOF'
#!/bin/bash
set -e

echo "=== SDLF Foundations Deployment ==="
echo "Organization: $ORGANIZATION_NAME"
echo "Environment: $ENVIRONMENT"
echo "Team: $TEAM_NAME"
echo "Dataset: $DATASET_NAME"
echo ""

# Load environment variables
source .env

# Deploy foundations stack
echo "Deploying SDLF Foundations..."
aws cloudformation deploy \
    --template-file nested-stacks/template-main.yaml \
    --stack-name sdlf-foundations-$ENVIRONMENT \
    --parameter-overrides file://parameters.json \
    --capabilities CAPABILITY_NAMED_IAM \
    --region $(aws configure get region)

echo "Foundations deployment completed!"
EOF

chmod +x deploy-foundations.sh

Troubleshooting

Common Issues:

1. Invalid parameter values

# Check parameter file syntax
python -m json.tool parameters.json

2. Account/Region mismatch

# Verify current AWS context
aws sts get-caller-identity
aws configure get region

3. Bucket name conflicts

# Check if bucket names are available
aws s3 ls s3://sdlf-demo-legislators-raw-dev-$(aws sts get-caller-identity --query Account --output text) 2>/dev/null || echo "Bucket available"

Verification

After completing this step, verify:

  • parameters.json file is configured correctly
  • config.json contains correct account and region
  • Environment variables are set
  • Resource naming follows SDLF conventions
  • Deployment script is ready
