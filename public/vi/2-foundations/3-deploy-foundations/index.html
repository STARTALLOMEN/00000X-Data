2.3 Triển khai CloudFormation Stack

Triển khai SDLF Foundations bằng AWS Console

Trong bước này, bạn sẽ triển khai SDLF Foundations sử dụng AWS CloudFormation Console.

1. Truy cập CloudFormation

  1. Đăng nhập AWS Console
  2. Tìm kiếm CloudFormation trong thanh tìm kiếm dịch vụ
  3. Click CloudFormation

2. Tạo Stack mới

  1. Click Create stack > With new resources (standard)
  2. Ở mục Specify template, chọn Upload a template file
  3. Click Choose file và chọn file sdlf-foundation.yaml đã chuẩn bị
  4. Click Next

3. Nhập tham số stack

  1. Đặt Stack name: sdlf-foundations-workshop
  2. Ở phần Parameters, nhập các giá trị:
    • Environment: workshop
    • ResourcePrefix: sdlf-workshop
    • VpcId: để trống (nếu dùng default VPC)
    • SubnetIds: để trống (nếu dùng default VPC)
    • LogLevel: INFO
  3. Click Next

4. Cấu hình stack options

  1. Có thể để mặc định các options
  2. Click Next

5. Xác nhận và tạo stack

  1. Kiểm tra lại thông tin
  2. Tích vào ô xác nhận quyền IAM nếu có
  3. Click Create stack

6. Theo dõi tiến trình triển khai

  1. Chuyển sang tab Events để theo dõi tiến trình
  2. Đợi trạng thái stack chuyển sang CREATE_COMPLETE

7. Troubleshooting

  • Nếu stack FAILED: Click vào stack để xem chi tiết lỗi ở tab Events
  • Kiểm tra lại template và tham số
  • Đảm bảo đủ quota và quyền truy cập

8. Sau khi triển khai thành công

  • Ghi lại tên các resource đã tạo (S3, Glue, IAM, KMS…)
  • Chuẩn bị cho bước xác minh triển khai

Lưu ý:

  • Không đóng tab khi stack đang tạo
  • Nếu gặp lỗi quota, hãy tăng quota trước khi thử lại
  • Xóa stack khi không còn sử dụng để tránh phát sinh chi phí
