Cấu hình credentials

Chúng ta sẽ cấu hình AWS credentials để AWS CLI có thể xác thực và truy cập vào tài khoản AWS của bạn.

Sau khi cài đặt AWS CLI, bạn cần cấu hình credentials để có thể sử dụng các lệnh AWS từ command line.

Lưu ý bảo mật: Không bao giờ chia sẻ Access Key và Secret Access Key của bạn với người khác. Đây là thông tin nhạy cảm có thể được sử dụng để truy cập vào tài khoản AWS của bạn.

Các phương pháp cấu hình credentials

Có 3 cách chính để cấu hình AWS credentials:

  1. AWS CLI configure - Cấu hình thủ công
  2. IAM Role - Sử dụng role (khuyến nghị cho EC2)
  3. Environment Variables - Biến môi trường

1. Sử dụng AWS CLI configure

Đây là cách phổ biến nhất để cấu hình credentials:

Bước 1: Lấy Access Key và Secret Access Key

  1. Đăng nhập vào AWS Console
  2. Đi tới IAM service
  3. Chọn Users từ menu bên trái
  4. Chọn user của bạn hoặc tạo user mới
  5. Chọn tab Security credentials
  6. Tạo Access Key mới

Nếu bạn đang sử dụng tài khoản root, có thể tạo Access Key trực tiếp từ IAM > Users > [Your Username] > Security credentials.

Bước 2: Cấu hình AWS CLI

Mở terminal/command prompt và chạy:

aws configure

Bạn sẽ được yêu cầu nhập các thông tin sau:

AWS Access Key ID [None]: AKIAIOSFODNN7EXAMPLE
AWS Secret Access Key [None]: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
Default region name [None]: us-east-1
Default output format [None]: json

Giải thích các thông tin:

  • AWS Access Key ID: Access Key bạn đã tạo
  • AWS Secret Access Key: Secret Access Key tương ứng
  • Default region name: Region mặc định (ví dụ: us-east-1, us-west-2, eu-west-1)
  • Default output format: Định dạng output (json, text, table)

2. Kiểm tra cấu hình

Sau khi cấu hình, kiểm tra xem credentials có hoạt động không:

aws sts get-caller-identity

Kết quả mong đợi:

{
    "UserId": "AIDACKCEVSQ6C2EXAMPLE",
    "Account": "123456789012",
    "Arn": "arn:aws:iam::123456789012:user/your-username"
}

3. Cấu hình nhiều profile (Tùy chọn)

Nếu bạn có nhiều tài khoản AWS hoặc muốn sử dụng nhiều role khác nhau:

aws configure --profile sdlf-workshop

Sau đó sử dụng profile:

aws s3 ls --profile sdlf-workshop

4. Sử dụng IAM Role (Khuyến nghị cho EC2)

Nếu bạn đang chạy trên EC2 instance, khuyến nghị sử dụng IAM Role:

  1. Tạo IAM Role với các policy cần thiết
  2. Gán role cho EC2 instance
  3. AWS CLI sẽ tự động sử dụng credentials từ role

5. Sử dụng Environment Variables

Bạn cũng có thể cấu hình credentials qua biến môi trường:

Windows:

set AWS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
set AWS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
set AWS_DEFAULT_REGION=us-east-1

Linux/macOS:

export AWS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
export AWS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
export AWS_DEFAULT_REGION=us-east-1

6. Kiểm tra quyền truy cập

Kiểm tra xem credentials có đủ quyền không:

# Kiểm tra quyền S3
aws s3 ls

# Kiểm tra quyền IAM
aws iam get-user

# Kiểm tra quyền CloudFormation
aws cloudformation list-stacks

Nếu bạn gặp lỗi “Access Denied”, có thể user của bạn không có đủ quyền. Hãy liên hệ với admin để được cấp thêm quyền.

7. Xóa credentials (Bảo mật)

Để xóa credentials đã lưu:

aws configure list
aws configure get aws_access_key_id
aws configure set aws_access_key_id ""
aws configure set aws_secret_access_key ""

Troubleshooting

Lỗi thường gặp:

  1. Invalid credentials: Kiểm tra lại Access Key và Secret Access Key
  2. Access Denied: Kiểm tra quyền của user
  3. Region not found: Kiểm tra tên region có đúng không

Kiểm tra cấu hình hiện tại:

aws configure list

Đến đây, chúng ta đã hoàn thành việc cấu hình AWS credentials. Tiếp theo chúng ta sẽ xác minh môi trường.
